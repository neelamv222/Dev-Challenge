Updating Table

This page is served from dev-challenge/site/index.html. To get set up, go into the dev-challenge directory and run npm install. To start the server, run npm start

It includes javascript code bundled starting at challenge/index.js and also a javascript library called stomp.js for connecting to a streaming pub/sub system over websockets. Within a short time from loading the page, it will connect over websockets to a stomp server that also runs when you run the development server.

It hasn't yet connected to the stomp server. If this state persists for a significant length of time, check

  • that you have run the development server by going into the db-updating-table directory and run it by running npm start
  • that there were no errors during the development server starting (e.g. perhaps port 8011 was taken)
  • that you are accessing it through that dev server, at http://localhost:8011 and that no errors are appearing in your browsers console.
If none of that works, then ask for help by emailing adam.iley@db.com.

Your Task

Your task is to show those currency pairs in a table with rows for each currency pair, which includes columns for at least the name, the current best bid price, current best ask price, the amount that the best bid last changed, and the amount the best ask price last changed. This data can be read out of the price updates that are sent via stomp. You should request the topic /fx/prices, to receive update messages with a body that looks like this: 

{
  "name": "usdjpy",
  "bestBid": 106.7297012204255,
  "bestAsk": 107.25199883791178,
  "openBid": 107.22827132623534,
  "openAsk": 109.78172867376465,
  "lastChangeAsk": -4.862314256927661,
  "lastChangeBid": -2.8769211401569663
}

To make the task more interesting, the table should be sorted (and remain sorted) by the column that indicates how much the best bid price last changed (lastChangeBid in the response data).
